## **Screenshots Documentation**

This folder contains all the screenshots related to the Student Wellness Assistant project. These images help demonstrate how the program works, how data is stored, and how different features behave during execution. The screenshots were taken directly from the terminal and CSV files while testing the program.

The contents of this folder are organized to match the sections of the project report.

## **Contents of This Folder**
## **1. Program Execution Screenshots**

   These images show the program running in the terminal:

   Main menu display

   User selecting different options

   Program flow while entering data

## **2. Daily Log Entry Screenshots**
 
Screenshots showing:

Input prompts for sleep, study hours, mood rating, steps and notes

Successful confirmation messages after saving a log

CSV file after adding new daily logs

## **3. Task Entry Screenshots**

Screenshots showing:

Entering a new task (task name, due date, importance)

Successful confirmation message

tasks.csv updated with new entries

## **4. Dashboard Output Screenshots**

These include:

Weekly averages (sleep, study, mood, steps)

Number of logs recorded

Burnout estimation

Urgent tasks sorted by priority

## **5. Exception / Error Handling Screenshots**

These images show how the program reacts to invalid inputs:

Wrong date format

Non-numeric sleep or study value

Missing or blank input

Incorrect menu choices

## **6. Code Snippet Screenshots**

To visually support the project report, this folder includes selected code snippet screenshots:

Daily log saving logic

Task management logic

Dashboard and burnout calculation

CSV reading/writing functions

These snippets highlight important parts of the program without showing the entire file.

## How to Use These Screenshots

These screenshots are meant to be:

Inserted into the project report

Displayed in the Testing or Results section

Linked inside the main README (optional)

Used as evidence of working features

Each screenshot helps demonstrate how the system behaves under normal use and error conditions.
